# Cornerstone 项目状态

**最后校对**: 2026-03-27  
**当前阶段**: 治理域一期可用，HTTP MCP 查询/建库/SSE 通知链路已扩展到主要业务变更，进入“回写闭环与跨系统联调”阶段  
**状态结论**: **核心业务可运行，治理闭环骨架已落地，HTTP MCP 已具备查询、建库、SSE 推送与 REST 变更联动能力，但真实双系统联调仍未完成**

## 1. 当前结论

截至 2026-03-24，项目不适合再用“98% / 100% 全部完成”描述。更准确的判断是：

| 领域 | 当前状态 | 说明 |
|---|---|---|
| 核心 CRUD 与权限 | **已实现** | 用户、组织、数据库、表、字段、记录、字段权限均已可用 |
| 插件 / 文件 / 统计 | **已实现** | 具备基础业务能力和 UI |
| Query DSL | **已实现并完成权限收口** | 白名单、自动过滤、管理员表限制已落地 |
| 治理任务中心 | **已实现** | 任务、审核、证据、评论、外链、详情 UI 均可用 |
| 入站治理事件 | **已实现** | 支持幂等处理和自动建单 |
| 审核通过后回写 | **已实现骨架** | 支持 apply、outbox、重试、死信，但仍待真实下游联调 |
| HTTP MCP | **已实现增强闭环** | 支持按当前 JWT 用户上下文查询、创建数据库，并支持 SSE 长连接、断线重放与业务变更主动通知 |
| 自动化测试 | **部分完善** | 后端单测已补，治理页 Playwright 回归已补；仍缺真实跨系统 E2E |
| 文档可信度 | **已修正** | `docs/API.md`、`docs/PROJECT-STATUS.md` 已按代码真值重写 |

## 2. 本轮已落地

### 2.1 后端

本轮已完成的关键修复与功能闭环：

- 治理任务权限已收紧到“创建者或 assignee（负责人）”可见可处理，避免系统自动任务被无关用户越权看到或修改。
- 审核通过后支持触发回写：
  - 新增审核回写入口 `POST /api/governance/reviews/:id/apply`
  - 新增治理 outbox 表 `governance_outbox_events`
  - 支持 `pending -> processing -> succeeded/failed/dead` 状态流转
  - 支持重试与后台 worker 轮询处理
- 入站集成事件幂等键已改为 `source_system + event_id`，避免不同来源系统使用同一事件号时互相冲突。
- Query DSL 做了权限收口：
  - 表/字段白名单校验
  - 敏感字段拦截
  - 管理类表访问限制
  - 自动补充基于用户权限的过滤条件
  - `select ["*"]` 会展开为允许字段，不会泄露 `users.password`
  - 修复了 JSON 路径查询和聚合计数不准确问题
- HTTP MCP 已接入主服务 `/mcp`：
  - 使用 JWT 登录态执行 tools
  - 当前提供查询与创建数据库能力
  - 增加了 `Origin` 校验，降低 DNS rebinding 风险
  - 已支持基于 SSE 的流式返回
  - `GET /mcp` 已支持长连接保活与数据库 / 表 / 字段 / 治理事件主动通知
  - SSE 改为按请求局部清除写超时，避免为支持流式连接而取消整站写超时保护
  - 已支持 `Last-Event-ID` 断线续传、`retry:` 提示与可配置 keepalive / replay buffer

### 2.2 前端

- 治理任务中心页面已补齐关键交互：
  - 任务列表筛选
  - 任务详情抽屉
  - 外部资源跳转链接
  - 评论与整改证据录入
  - 审核发起、审批、回写触发
  - 审核模板 JSON 快速填充
- 嵌入前端资源的脚本已从 CJS 改为 ESM，前端 lint 可通过。
- 已重建嵌入到 Go 服务中的前端静态资源。

### 2.3 自动化测试

## 5. 仍然存在的真实缺口

以下不是“文档没更新”的假问题，而是当前项目仍未闭环的真实事项：

| 缺口 | 当前状态 | 风险 |
|---|---|---|
| `cornerstone -> fuckcmdb` 真实回写联调 | **未完成** | 下游接口路径、鉴权、响应格式仍可能与当前假设不一致 |
| `fuckcmdb -> cornerstone` 双端集成验收 | **未完成** | 需要真实事件流验证自动建单和重复事件处理 |
| HTTP MCP 业务通知仍不完整 | **部分完成** | 已扩展到数据库/表/字段/治理任务/审核，但记录、文件、插件等事件仍未接入 |
| 治理域全链路 E2E | **未完成** | 当前新增的是 mock UI 回归，不覆盖真实数据库和真实后端 |
| 历史 Playwright 套件稳定性 | **未复核** | 现有权限类 E2E 仍偏依赖真实环境与测试账号 |
| 文档与测试资产清理 | **部分完成** | 两份主文档已修；其余历史 README / 报告仍可能滞后 |

## 6. 后续建议

建议下一步按优先级推进：

1. 完成 `fuckcmdb` 对接联调，确认回写接口、鉴权头、失败码和响应体契约。
2. 做一条真实的治理闭环验收用例：
   `fuckcmdb` 产生事件 -> `cornerstone` 自动建单 -> 人工审核 -> 回写成功。
3. 若 HTTP MCP 要对接更多客户端，继续扩展 `GET /mcp` 的通知类型、重连恢复和更完整的 Streamable HTTP 约定。
4. 清理或重写历史 Playwright 套件与 `frontend/e2e/README.md`，避免继续误导测试范围。
5. 若治理域继续扩展，再补 SLA、任务看板、批量处理和更细的权限模型。
